Flask Framework Boilerplate

A clean and organized structure for Flask projects

Folder Structure

/project-root ├── /app │ ├── __init__.py │ ├── config.py │ ├── /routes │ │ ├── __init__.py │ │ ├── auth.py │ │ ├── users.py │ │ └── items.py │ ├── /models │ │ ├── __init__.py │ │ ├── user.py │ │ └── item.py │ ├── /services │ │ ├── __init__.py │ │ ├── auth_service.py │ │ ├── user_service.py │ │ └── item_service.py │ ├── /templates │ │ ├── base.html │ │ ├── index.html │ │ └── login.html │ ├── /static │ │ ├── css/ │ │ ├── js/ │ │ └── images/ │ ├── /db │ │ ├── __init__.py │ │ ├── database.py │ │ └── migrations/ │ ├── /utils │ │ ├── __init__.py │ │ ├── helpers.py │ │ ├── jwt_handler.py │ │ └── validators.py │ ├── /tests │ │ ├── __init__.py │ │ ├── test_auth.py │ │ ├── test_users.py │ │ └── test_items.py │ └── .env ├── .gitignore ├── requirements.txt ├── wsgi.py ├── Dockerfile ├── docker-compose.yml └── README.md

Key Features

  • Follows Flask's best practices
  • Modular structure using Blueprints
  • Separation of concerns for improved scalability
  • Environment variable management for secure credentials
  • Includes folders for static files, templates, and tests

Getting Started

Clone this boilerplate and run the following commands:

pip install -r requirements.txt
python wsgi.py
